# SellGeneralAdminCost_Allocation_Cmd.py 仕様（0005_PJサマリ 出力ファイル名変更）

## 目的
`SellGeneralAdminCost_Allocation_Cmd.py` における `0005_PJサマリ` 系の出力について、
`step0006` 名義で出力しているファイル名を `step0007` 名義に変更する。

## 対象範囲
- 対象: `0005_PJサマリ` の「単・累_損益計算書」出力ファイル名。
- 非対象:
  - 入力ファイル名（`step0005`）
  - 出力内容（TSV中身の計算ロジック）
  - `0004_PJサマリ` や `0001/0002/0003` 系など他プレフィックスの命名

## 現行仕様（変更前）
`step0005` を入力として読み込み、次の `step0006` ファイルを生成している。

- `0005_PJサマリ_step0006_単・累_損益計算書_YYYY年MM月-YYYY年MM月.tsv`

例:
- `0005_PJサマリ_step0005_単・累_損益計算書_2025年04月-2025年10月.tsv` を入力
  - `0005_PJサマリ_step0006_単・累_損益計算書_2025年04月-2025年10月.tsv` を出力
- `0005_PJサマリ_step0005_単・累_損益計算書_2025年09月-2025年10月.tsv` を入力
  - `0005_PJサマリ_step0006_単・累_損益計算書_2025年09月-2025年10月.tsv` を出力

## 変更後仕様（要求仕様）
`step0005` を入力として読み込み、出力ファイル名を `step0007` に変更する。

- 変更後出力名: `0005_PJサマリ_step0007_単・累_損益計算書_YYYY年MM月-YYYY年MM月.tsv`

例:
- `0005_PJサマリ_step0005_単・累_損益計算書_2025年04月-2025年10月.tsv` を入力
  - `0005_PJサマリ_step0007_単・累_損益計算書_2025年04月-2025年10月.tsv` を出力
- `0005_PJサマリ_step0005_単・累_損益計算書_2025年09月-2025年10月.tsv` を入力
  - `0005_PJサマリ_step0007_単・累_損益計算書_2025年09月-2025年10月.tsv` を出力

## 振る舞い要件
1. `step0005` 入力の検出条件は従来と同一であること。
2. 生成対象は従来の `step0006` 相当の中身を維持し、**ファイル名のみ** `step0007` に変更すること。
3. 同一実行内で複数期間（例: `2025年04月-2025年10月`, `2025年09月-2025年10月`）を処理する場合でも、すべて `step0007` 名義で出力されること。
4. 他系列のステップ番号運用（`0004_PJサマリ` など）には影響を与えないこと。

## 受け入れ基準
- 以下の入力が存在する場合:
  - `0005_PJサマリ_step0005_単・累_損益計算書_2025年04月-2025年10月.tsv`
  - `0005_PJサマリ_step0005_単・累_損益計算書_2025年09月-2025年10月.tsv`
- 実行後、以下が生成されること:
  - `0005_PJサマリ_step0007_単・累_損益計算書_2025年04月-2025年10月.tsv`
  - `0005_PJサマリ_step0007_単・累_損益計算書_2025年09月-2025年10月.tsv`
- かつ、上記ケースにおいて `0005_PJサマリ_step0006_単・累_...` は新規生成されないこと。

## 補足
- 本仕様書は「ファイル名変更要求」の明文化を目的とし、実装変更は含まない。
